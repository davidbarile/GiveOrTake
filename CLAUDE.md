# GiveOrTake Claude Code Guide

## Project summary
GiveOrTake is a guest-first multiplayer pod game prototype.

## Architecture
- Monorepo using pnpm workspaces.
- `apps/web`: Next.js 15 + React + TypeScript + Tailwind, runs on port `3001`.
- `apps/api`: NestJS API, runs on port `4000`, global API prefix `/api`.
- `prisma/schema.prisma`: PostgreSQL data model.
- Redis is required for runtime features.
- Socket.IO realtime namespace is `/ws`.

## Product rules
- Preserve guest-first onboarding: users should be able to open the app, get an autogenerated guest identity, choose/join a pod, and play without upfront account creation.
- Do not add mandatory auth before gameplay unless explicitly requested.
- Claimed accounts/custom pod creation can remain gated, but the core quickstart flow must stay guest-accessible.

## Key commands
Use arm64 Node >=22.13 and pnpm >=9.

```bash
pnpm install
pnpm -r build
pnpm -r test
bash scripts/smoke-api.sh
```

Local services:
```bash
docker compose -f infrastructure/docker/docker-compose.yml up -d
```

Prisma schema push:
```bash
DATABASE_URL="postgresql://devuser:devpass@localhost:5432/give_or_take" pnpm exec prisma db push --schema=prisma/schema.prisma
```

Development:
```bash
pnpm --filter @got/api dev
pnpm --filter @got/web dev
```

Production starts:
```bash
pnpm --filter @got/api start
pnpm --filter @got/web start
```

## Deployment target
Hostinger VPS / plain Ubuntu:
- Nginx public reverse proxy
- web app on `127.0.0.1:3001`
- API on `127.0.0.1:4000`
- PostgreSQL and Redis either on-VPS or managed external services
- PM2 for Node process supervision

Deployment docs:
- `deploy/hostinger-vps/README.md`
- `deploy/hostinger-vps/README-plain-ubuntu-no-domain.md`

## Safety and workflow
- Before refactors, inspect `git status` and keep changes scoped.
- Do not commit secrets or real `.env` files.
- Update `.env.example` or deploy env examples when introducing env vars.
- After any runtime/deployment change, update the relevant file under `deploy/hostinger-vps/`.
- Run the narrowest relevant test/build after edits, then report real output.

## Common pitfalls
- Do not treat this as a static site; it needs web, API, Postgres, Redis, and WebSocket routing.
- Raw-IP HTTP deployments need `COOKIE_SECURE=false`.
- Public Next.js env vars must be set before `pnpm -r build` because they are baked into the frontend bundle.
- Keep API and websocket URLs aligned with the external origin.
