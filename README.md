# Give or Take Pods

Playable MVP prototype for the guest-first **Give or Take** Pods spec.

## Stack

- Next.js 15 + React + TypeScript + Tailwind frontend
- NestJS API
- PostgreSQL + Prisma
- Redis
- Socket.IO groundwork for realtime pod updates

## Guest-first flow

Users do **not** need accounts to start playing:

1. open the app
2. a guest `Player` and `DeviceSession` are bootstrapped automatically
3. an autogenerated username is assigned
4. user chooses Quickstart / a public pod and plays
5. user can later save progress by claiming the account

Guests may join and play immediately. Claimed accounts are required for custom pod creation and future pod-slot ownership / purchases.

## Run locally

```bash
export PATH="$HOME/.nvm/versions/node/v26.4.0/bin:$PATH"
cd /Users/davidbarile/Documents/HermesWorkspace/GiveOrTake
pnpm install

docker compose -f infrastructure/docker/docker-compose.yml up -d
DATABASE_URL="postgresql://devuser:devpass@localhost:5432/give_or_take" node node_modules/prisma/build/index.js db push --schema=prisma/schema.prisma

# terminal 1
cd apps/api && node dist/main.js

# terminal 2
cd apps/web && pnpm dev
```

Open: <http://localhost:3001>

## Verify

```bash

```

The smoke test verifies:

- guest session bootstrap
- autogenerated username
- Quickstart Classic pod join
- TAKE gameplay action

- guests are blocked from custom pod creation until account claim
